## ⚠️ 发现的问题及修复

### 问题1: CandyBox2 存档清除不完整
**位置**: `src/components/CandyBox2Game.vue:382-387`

**问题描述**:
当存档无效时，清除了 uTools 存档，但没有清除 localStorage 中的 pending 数据，可能导致加载错误的槽位。

**影响**: 中等
- 可能导致用户看到错误的槽位数据
- 但不会导致游戏崩溃或白屏

**修复方案**:
```javascript
} catch (parseError) {
  console.error('[CandyBox2] 解析存档失败，使用默认槽位:', parseError)
  // 清除无效的存档
  window.utools.dbStorage.removeItem(saveKey)
  // 同时清除 localStorage 中的 pending 数据
  localStorage.removeItem('candybox2_pending_slot')
  localStorage.removeItem('candybox2_pending_data')
  initialSlot = '1'
}
```

### 问题2: vConsole 在 uTools 中可能影响性能
**位置**: `src/main.js:14-34`

**问题描述**:
vConsole 在所有环境都会初始化，包括 uTools 生产环境。虽然默认隐藏，但仍占用内存（287KB）。

**影响**: 低
- 内存占用可接受
- 但可以考虑在生产环境禁用

**优化建议**:
```javascript
// 只在开发环境或特定调试模式下启用 vConsole
const enableVConsole = import.meta.env.DEV || localStorage.getItem('debug') === 'true'

if (typeof window !== 'undefined' && enableVConsole) {
  // ... vConsole 初始化代码
}
```
